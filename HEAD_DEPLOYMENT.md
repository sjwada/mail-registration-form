# 開発ワークフロー補足：HEADデプロイメントについて

## HEADデプロイメント（`/dev` URL）とは

GASのWebアプリデプロイメントには2種類あります：

### 1. HEADデプロイメント（開発用）
- **URL**: `/dev`で終わる
- **特徴**: `clasp push`で即座に反映される（再デプロイ不要）
- **用途**: 開発・テスト
- **アクセス**: 自分のみ

### 2. バージョン付きデプロイメント（本番用）
- **URL**: `/exec`で終わる
- **特徴**: 特定のバージョンに固定、更新には再デプロイが必要
- **用途**: 本番環境
- **アクセス**: 全員（または指定したユーザー）

## 開発ワークフロー

```bash
# 1. ローカルでコードを編集
# （VS Codeなどで src/ 配下のファイルを編集）

# 2. GASにプッシュ
clasp push

# 3. /dev URLで即座にテスト（再デプロイ不要！）
# ブラウザで /dev URL にアクセス
# 変更が即座に反映されています
```

## 重要なポイント

✅ **`/dev` URLは`clasp push`で即座に更新される**  
✅ **再デプロイは不要**  
✅ **開発中は常に`/dev` URLを使用**  
✅ **本番公開時のみ、バージョン付きデプロイメント（`/exec`）を作成**

## 本番デプロイメントの作成（テスト完了後）

テストが完了し、本番公開する準備ができたら：

1. GASエディタで「デプロイ」→「新しいデプロイ」
2. 種類：「ウェブアプリ」
3. 説明：「本番環境 v1.0」
4. **新しいバージョン**を作成（説明を入力）
5. アクセスできるユーザー：「全員」
6. デプロイ

これにより、`/exec`で終わる本番URLが作成されます。このURLは保護者に配布します。

## まとめ

- **開発中**: `/dev` URL を使用（`clasp push`で即座に反映）
- **本番公開**: `/exec` URL を使用（バージョン固定、安定版）
